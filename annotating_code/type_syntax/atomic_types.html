    アトミック型¶

    アトミック型は、Psalmで使用されるすべての型情報の基本的な構成要素です。複数のアトミック型は、ユニオン型やインターセクション型と組み合わせることができます。Psalmでは、docblock構文で多くの異なる種類のアトミック型を表現できます。

    注意: 以下のリストの各型をクリックすると、すべてのアトミック型の詳細なドキュメントと使用例を表示できます。

    • スカラー型
      • bool
      • int
      • float
      • string
      • int-range<x, y>
      • int-mask<1, 2, 4>
      • int-mask-of<MyClass::CLASS_CONSTANT_*>
      • class-string と class-string<Foo>
      • trait-string
      • enum-string
      • callable-string
      • numeric-string
      • literal-string
      • literal-int
      • array-key
      • numeric
      • scalar
    • オブジェクト型
      • object
      • object{foo: string}
      • Exception, Foo\MyClass と Foo\MyClass<Bar>
      • Generator
    • 配列型
      • array<int, string>
      • non-empty-array
      • string[]
      • list & non-empty-list
      • list<string>
      • array{foo: int, bar: string} と list{int, string}
      • callable-array
    • 呼び出し可能型
    • 値型
      • null
      • true, false
      • 6, 7.0, "forty-two" と 'forty two'
      • Foo\Bar::MY_SCALAR_CONST
    • ユーティリティ型
      • (T is true ? string : bool)
      • key-of<T>
      • value-of<T>
      • properties-of<T>
      • class-string-map<T of Foo, T>
      • T[K]
      • 型エイリアス
      • 変数テンプレート
    • その他の型
      • iterable<TKey, TValue>
      • void
      • resource
      • closed-resource
    • トップ型とボトム型
      • mixed
      • never

    ユーザーノートについて

    PHPマニュアルのように、マニュアルを補足する情報の共有を目的としてます。

    • 掲示板ではありません。議論、質問、別のノートへの返信はできません。
    • バグや改善要望、または翻訳の Issue は、該当する GitHub リポジトリで報告してください。
    • ノートは事前の通知なく削除される場合があります。
    • Giscussのコメントシステムを流用しています。そのためノートとしては最適化されてないことをご了承ください。
    • 保証されていません。投稿したノートが消失する事やノート機能全体が閉鎖する可能性があります。
    • お願い：いいノートにはup voteを、この規約に違反しているノートを見つけた場合は、優しくdown voteしてください。
